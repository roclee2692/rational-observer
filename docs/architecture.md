# Rational Observer — 架构设计 v1.1 (施工版精简)

> 定位:常驻的纯理性「观察者版本的你」。**只观察/记忆/提醒/追踪,不决策、不预测、不下单。**
> 目标:扩展认知带宽 + 通过预测追踪迭代判断力。原则:数据无损 / 记忆分层 / 决策留痕 / 认知可追溯。

## 三个锁定决策
1. **观察者 ≠ 决策者**:日报里不出现"买/卖"建议,只给"事实 + 配比状态"。决策永远你做。
2. **事实 vs 理解 分离**(最重要):
   - 事实(价格/指标/成交量)→ 永不压缩 → 时序库 L0
   - 理解(新闻解读/叙事/分析)→ 可分层压缩 → 向量库 L1-L3
   - 混在一起是 90% agent 记忆系统失败的根源。
3. **为"跑三年"设计**:从第一天解决记忆膨胀、检索退化、上下文爆炸。

## 资产注册表 (第三节) —— 单一事实来源
`config/assets.yaml` 定义整个资产宇宙。采集/存储/分析/配比都从它读取。
- 加资产 = 改 YAML,零代码。`enabled:false` = 停采但保留历史。
- **持仓资产** target_pct>0(进库✓ 配比✓ 再平衡✓);**观察资产** target_pct=0(进库✓ 可分析✓ 不计入组合)。
- 启动校验:持仓 target 总和必须=100%;coingecko 必填 coin_id,yfinance 必填 ticker。

## 五层分层记忆 (第四节,系统灵魂)
| 层 | 存储 | 内容 | 策略 |
|---|---|---|---|
| L0 原始 | DuckDB | 每日 OHLCV + 宏观指标 | 永不压缩/删除 |
| L1 日志 | 向量库 | 每日简报(~200字) | 全文保留 90 天 |
| L2 周报 | 向量库 | 本周主线+关键变化(~300字) | 保留 1 年 |
| L3 主题 | 向量库 | 长期叙事线(如"美元走弱进程") | 永久 |
| L4 台账 | SQLite | 预测+理由+置信度+结果+复盘 | 永久 |
检索路由:精确数值→L0 / 近期→L1 / 中期→L2 / 长期叙事→L3 / 预测复盘→L4 / 复合→多层组合。

## 数据库 (第五节)
- L0 时序:**DuckDB**(单文件、列式、聚合快)。表:daily_prices / macro_indicators / portfolio_snapshots。
- 理解:**Chroma/LanceDB**,每条带元数据(level/date/assets/sentiment)做混合检索。
- 结构化:**SQLite**。表:predictions / actions。

## Agent 调度 (第七节, Hermes + 飞书)
- **Job1 每日采集**(收盘后,Haiku):抓数据→L0,生成简报→L1,查异常。
- **Job2 异常检测**(只在触发时推飞书):配比偏离±5% / VIX>30 / 单日波动>3% / 预测到期。宁缺毋滥。
- **Job3 周报/月报**:L1→L2→L3 压缩 + 图表 + 推送。
- **Job4 查询路由**:简单→Haiku 查 L0;复杂→检索 L3+L0 分发强模型;预测→L4。

## 预测追踪 (第八节, 认知迭代核心)
飞书录预测→写 L4(prediction/reasoning/confidence/target_date)。到期自动从 L0 取实际走势→判对错→LLM 复盘→准确率曲线(按类别/置信度分)。把"痛苦+反思=进步"量化。

## 统计分析 (第六节)
相关性矩阵 / 配比漂移 / 滚动波动率+回撤 / 归一化对比 / 宏观叠加 / 预测准确率。
可视化:matplotlib/plotly→飞书推送(起步)→ Streamlit(进阶)。**图表用英文标签**(防中文缺字)。

## 路线图 (第九节)
Phase0 MVP(assets.yaml+L0+采集+日报) → Phase1(向量L1+警报+查询) → Phase2(分析+可视化+周报) → Phase3(L2/L3+L4预测+复盘)。**时间盒 40-50h / 4-6 周**;系统是工具,认知积累才是目的。

## 关键风险 (第十一节)
记忆膨胀(严格分层压缩)/ 过度工程(时间盒)/ 警报疲劳(阈值保守)/ 系统替代思考(只给事实不给信号)。
